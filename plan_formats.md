# Format planów – rekomendacja

## Markdown (.md)
- checklisty, specyfikacje, instrukcje deploy
- idealne do Git (diffy, historia)

## YAML (.yml)
- taski do automatyzacji/importu (Jira/Trello)
- łatwe do parsowania skryptem

## XLSX/CSV
- payroll, koszty, eksporty

**W praktyce:** trzymasz `docs/*.md` + `docs/plan.yml` + eksporty w `exports/`.
